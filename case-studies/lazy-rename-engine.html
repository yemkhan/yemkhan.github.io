01

/ The problem

Naming game ads is its own full-time job.

Mobile game ad creatives live and die by their filenames. When comps get saved as final.mp4, final_v2.mp4, FINAL_real.mp4, ultra_final_USE_THIS.mp4, two things break at once. Meta's algorithm starts treating near-duplicate creatives as the same asset — impressions and clicks pile onto whichever variant the system picked first, and the rest of the batch silently dies. Worse, when a creative actually performs and you want to iterate on it, there is no way to trace the file back to the artist, the campaign, or the source project. You're left scrolling a Finder full of auto-save 47.mp4 trying to guess which one shipped.

The pain wasn't the convention itself — it was applying it. I'd sit inside After Effects double-clicking comp names one by one, retyping the same six-segment string twenty times, fat-fingering an underscore, and then re-rendering because the queue exported a typo. Multiply that by a busy week and the naming step alone was eating half a day.

02

/ The insight

The names follow a formula. Formulas are scripts.

Every name I was typing by hand was just structured fields glued together with underscores: a game code, an incrementing serial number, an artist initial, a project sub-name + concept colorway, the motion + level, and an aspect-ratio suffix. If a human could produce the string by reading those fields, an ExtendScript could produce it faster — and never typo. Render the file as MI_520_MK_Stargazer-Red_Motion-Lv2_1080x1920_30s and suddenly every downstream system — Meta, the render sheet, the artist lookup — knows exactly what it's looking at.

03

/ The build

One panel. Nine fields. Per-row Copy and Replace.

I wrote Lazy Rename Engine™ as a single ExtendScript file that opens a ScriptUI panel inside After Effects. You fill the fields once, hit Generate Names, and the panel lists every output line with its own Copy and Replace button — copy to paste into a render sheet, or replace to rewrite the corresponding CompItem in place.

The smart bit is the serial range: the panel reads a start and an end serial number and grows the Concept inputs to match the count. Set 520 → 523 and you get four concept slots; bump the end to 527 and four more appear without re-opening the panel.

  • Dynamic concept slots. endSerial − startSerial + 1 drives how many Concept fields render. No fixed cap.

  • Suffix presets + custom override. vert30, vert15, square, FB 1080×1350, landscape — or type your own resolution-duration string.

  • Per-row Copy. Each generated name has its own Copy button, so a single row can feed a single render entry without touching the others.

  • Per-row Replace. Replace walks the project, finds the matching CompItem (by index or selection) and rewrites .name in place — wrapped in an undo group, so one Cmd+Z reverts the whole batch.

04

/ The output

One name per serial, ready to render.

Once names are generated, the workflow forks. Most days I hit Replace on each row so the comps themselves get renamed — then the After Effects render queue (or Adobe Media Encoder) inherits the new names automatically, and the MP4s land on disk already matching the convention. No filesystem rename pass. No spreadsheet reconciliation.

05

/ The outcome

Boring work, automated. Boring work, done.

The first time I ran v5 on a real campaign, fifty-two comps got renamed in under a second. The rest of the afternoon I spent on actual motion design instead of shepherding underscores. That's the whole pitch.

The script is open source — pull it, fork the field layout to match your studio's convention, and stop typing the same string twenty times.
