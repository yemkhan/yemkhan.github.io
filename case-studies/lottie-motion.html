After Effects → Lottie Interaction Design

Animating motion states in After Effects, exporting through Lottie, and wiring them into interactive web components — micro-motion as a real design surface.

▶ Try the Interactive Lottie →

↑ State-machine Lottie — tap the animation or use the controls

01

/ The pipeline

From timeline to interactive web, one export at a time.

The web doesn't need a video file. It needs a vector animation that can react — to scroll, to hover, to a click, to a state change. That's what Lottie is for. The trick is treating After Effects as a component authoring tool, not a video editor: every layer is a named state, every marker is an interaction trigger, every Bodymovin export is a designed contract with the front-end.

02

/ The build

Animate states. Export as .lottie. Drop in.

The animation above ships as a single .lottie file with a Lottie Creator state machine inside it. The web player loads StateMachine1, then routes button and canvas clicks into named inputs like vanilla, straw, and reset so the animation changes state instead of just looping like a video.

  • One file, every browser. The .lottie archive bundles the animation JSON, fonts, and any inline assets — no missing-asset edge cases.

  • State-driven, not frame-driven. Layers and markers are named for intent (idle, hover, pressed, success) so the front-end can play segments instead of guessing frame numbers.

  • Interactive surface. The dotLottie player exposes events for hover, click, segment-end — letting micro-motion drive real UI behaviour.

  • Tiny payload. Vector + JSON means a state animation that would be a 2 MB MP4 lands as a 30 KB asset.

03

/ The outcome

Motion that lives inside the product, not next to it.

The classic motion-design failure mode is the gorgeous reel that never makes it into the shipping product. Lottie closes that gap — the animation you approved in After Effects is literally the animation that loads in production. No "we'll re-create it in code." No fidelity loss. Just the file.
